# Wheel Strategy Dashboard — Android App

An Android build of the **Wheel Strategy Dashboard** (Cash Secured Puts & Covered Calls tracker). The web dashboard is wrapped in a native Android shell using [Capacitor](https://capacitorjs.com/), so it installs and runs like a normal app while keeping the exact same UI and features.

All trade data is stored **locally on the device** (browser `localStorage` inside the app's WebView). Nothing is sent to a server — a Google Sheets URL is used only for optionally pulling in base values.

---

## 📲 Install the app (no build tools needed)

1. Open the repo's **[Actions](../../actions)** tab.
2. Click the most recent **Build Android APK** run (green checkmark).
3. Under **Artifacts**, download **`wheel-dashboard-debug-apk`** (a `.zip`).
4. Unzip it to get **`app-debug.apk`**.
5. Copy the APK to your Android phone and tap it to install.
   - You'll need to allow **"Install unknown apps"** for your file manager / browser the first time (Android will prompt you).

> This is a **debug** APK — perfect for personal use and testing. It is not signed for the Google Play Store.

## ✨ Features

- Full Wheel Strategy dashboard (summary cards, charts, options & stock tables, open positions, DTE tracking).
- Works **offline** — Chart.js is bundled locally, no internet required to use it.
- **Backup / Restore** and **CSV export** work via Android's native Share sheet (save to Drive, email, files, etc.).
- Data persists on the device between launches.

## 🔄 Updating the app after changing the dashboard

The source HTML lives in [`www/index.html`](www/index.html). To ship an update:

```bash
# 1. Replace www/index.html with your latest dashboard HTML
#    (keep the <script src="chart.umd.min.js"> line and the native-shim
#     <script> block at the bottom — see notes below)

# 2. Commit and push
git add www/index.html
git commit -m "Update dashboard"
git push
```

Pushing to `main` automatically triggers a new APK build in GitHub Actions. Download the new artifact and reinstall.

### Two things to preserve when replacing the HTML
1. **Chart.js reference** — near the top, use the local copy:
   ```html
   <script src="chart.umd.min.js"></script>
   ```
   (not the `cdnjs.cloudflare.com` URL, so charts work offline).
2. **Native download shim** — the `<script>` block just before `</body>` that reroutes file downloads through Capacitor's Share sheet. Without it, Backup/Restore and CSV export do nothing inside the app.

## 🛠 Building locally (optional)

Requires Node.js, JDK 17, and the Android SDK.

```bash
npm install
npx cap sync android
cd android
./gradlew assembleDebug        # Windows: gradlew.bat assembleDebug
# APK output: android/app/build/outputs/apk/debug/app-debug.apk
```
